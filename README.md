# オリパ商品設計ツール

競合オリパを参考に、在庫から商品構成を自動生成するツールです。

## 起動方法

ターミナルで以下を実行：

```bash
cd /Users/risa/oripa-designer
source venv/bin/activate
streamlit run app.py
```

ブラウザで `http://localhost:8501` が自動で開きます。

## 基本フロー

```
① 新規設計タブで参考競合を選ぶ
    ↓
② パラメータ入力（総口数・1回価格・粗利率）
    ↓
③ モード選択（案X: 1枚あたり目標相場 / 案Y: 還元率+配分比率）
    ↓
④ 等構成を入力
    ↓
⑤ プレビュー or 保存（予約中として仮引当）
    ↓
⑥ 商品一覧タブで 承認（販売中へ）／解除（ボツへ）
    ↓
⑦ 販売終了後、完売ボタンで在庫数量を減算
```

## 状態遷移と在庫数量の動き

在庫シートには以下の数量列があります：
- **数量**: 物理在庫
- **予約中数量**: 仮引当中の枚数
- **販売中数量**: 販売中の枚数
- **残数量** = `数量 - 予約中 - 販売中` ＝新規に引当可能な枚数

| 操作 | 商品ステータス | 予約中 | 販売中 | 数量 | 残 |
|---|---|---|---|---|---|
| 保存 | → 予約中 | +N | - | - | -N |
| 承認 | 予約中→販売中 | -N | +N | - | - |
| 解除 | 予約中→ボツ | -N | - | - | +N |
| 完売 | 販売中→完売 | - | -N | -N | - |

**数量が10のBOXを3個予約しても、残7個は別商品に使えます**（数量ベース管理）

## スプシの構造

**在庫管理スプシ**に以下が追加されます：
- **PSA10在庫登録** / **ボックス在庫記録** に3列追加
  - 引当ステータス
  - 引当先商品ID
  - 引当日時
- **商品設計** タブ（商品サマリ）
- **商品設計明細** タブ（各商品の等・カード明細）

## ファイル構成

```

├── sheets_client.py    # Google Sheets API ラッパー
├── inventory.py        # 在庫読込・引当
├── research.py         # 競合リサーチ読込
├── designer.py         # マッチングロジック
├── operations.py       # 承認/解除/完売

```

## トラブル時

- **API接続エラー**: `credentials.json` のサービスアカウントがスプシに共有されているか確認
- **在庫読込0件**: 在庫シートの数量・相場が空になっていないか確認
- **競合が見つからない**: リサーチスプシを再確認。`load_all_references()` は30秒キャッシュ

## 運用のヒント

- 担当者から修正依頼が来たら：商品一覧から「解除」→再度設計→「保存」の流れで対応
- 同じカードを別の商品に使いたい場合：元商品を「解除」してから新商品を作成
- 販売停止したいが在庫復活したい場合：「解除」を使う（ボツ扱いになるが在庫は戻る）
