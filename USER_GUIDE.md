## ⚠️ 最初に必ず読むこと

このツールは **Googleスプレッドシート「ポケモン在庫管理」と常時連動** しています。

- ツールで操作した内容は **即座にスプシに書き込まれます**
- 逆にスプシを直接編集すると、ツール側の表示も変わります
- スプシを手で編集する時の注意は本ガイド末尾「スプシを触るときのルール」を必ず読んでください

## 1. このツールでできること

競合の人気オリパを参考にしながら、自社の在庫から自動でカードを選定して商品設計ができます。勘に頼らず、データに基づいた商品作りが可能です。

### 主な機能

- 🎴 **競合商品を丸ごとコピー**: トレカセンター約9,000件・DOPA全件のDBから、URL貼り付けで景品明細まで自動展開
- ✨ **リライト商品案100件**: 完売実績商品をベースに、利益率45〜50%になるよう自動調整された下書きが100件常備
- 🔍 **スニダンで相場自動取得**: PSA10直近販売価格・パック新品最安価格を自動取得。**毎朝06:00に自動更新**
- 🖼 **画像でカード自動照合**: PSA10カードの画像から在庫DB→スニダンURLを自動紐付け
- 📦 **在庫と売上を同期**: 予約中→販売中→完売の状態遷移で、物理在庫を自動で減算
- 🔄 **改善提案**: 新カード仕入時に「これに差し替えた方が良い」を自動検出
- 📊 **リアルタイム利益計算**: 発送限別コスト・上乗せ差分・実利益率を画面上部に固定表示

## 2. ログイン

1. 共有されたURLをブラウザで開く
2. パスワードを入力
3. **「ログイン」** ボタン

> パスワードが分からない場合は運用担当者に確認してください。

ログイン後、上部にサイドバーで **作業者名** を入力してください（誰が保存したか記録に残るため）。

## 4. 商品を作る3つのやり方（重要）

### やり方A: リライト商品案から作る（**新メンバーはまずコレ**）

1. **✨リライト商品案タブ** を開く
2. 100件の下書きから作りたいNo.を選ぶ（サムネ・単価・総口数・利益率がリスト表示）
3. **フィルタ** で絞込
   - タイトル / No. で検索
   - 利益率フィルタ（45-50%達成 / 不足 / 過剰 / 赤字）
   - 調整ステータス（未着手 / 調整中 / 完成 等）
4. 行をクリック → **「📋 景品設計に転送」** ボタンをクリック
5. **📋景品設計タブ** に自動遷移 → 「📥 読み込み」ボタン
6. 明細が自動展開されるので、必要に応じてカード相場や本数を調整
7. **「💾 保存」** で確定

### やり方B: 競合URLをコピペする

1. **📋景品設計タブ** を開く
2. 「🔗 商品URLを貼って即取込」欄に URL を貼る
   - `https://japan-toreca.com/oripa/...`
   - `https://dopa-game.jp/pokemon/gacha/...`
3. **「📥 URLから取込」** をクリック
4. タイトル / 単価 / 総口数 / 賞構成 / カード明細 が全自動展開
5. 必要に応じて調整して「💾 保存」

### やり方C: ゼロから設計（応用）

1. **📝新規設計タブ** を開く
2. 左サイドバーで参考競合を検索・選択
3. 販売パラメータ（総口数・1回価格・目標粗利率）を入力
4. モード選択（案X: 目標相場直指定 / 案Y: 還元率＋等級配分比率で自動計算）
5. 各等の当たり数・目標相場を入力
6. **「🔍 自動提案」** → 在庫から目標相場に最も近いカードが自動選定
7. 手動調整（❌削除 / 候補から➕追加）
8. **「💾 この内容で保存」**

## 5. 各タブの詳細

### 📝 新規設計タブ

**用途**: 通常オリパ（コイン還元なし・ポイント式）をゼロから設計

**流れ**:

1. サイドバー「① 参考競合を選ぶ」で検索
2. 「② 在庫モード」を選択
   - **在庫連動**: 残数量がある在庫だけ選択、保存で予約中に
   - **無在庫**: 全カード選択可、在庫予約は発生しない（販売決定後に仕入れ前提）
3. 「③ 販売パラメータ」（総口数・1回価格・目標粗利率）を入力
4. モード選択（案X / 案Y）と等構成入力
5. 「🔍 自動提案」→ 手動調整
6. 「💾 保存」

**画面上部に固定表示される計算結果**:
- 売上 / 原価合計 / 実還元率 / 実粗利率
- 警告バッジ（🔴 致命 / 🟡 注意 / 🔵 情報）

---

### 🎰 限定ガチャタブ

**用途**: DOPA型（ポイント還元・課金者限定）の限定ガチャを設計

- 「引く権利の事前課金額」を入力できる（有料ガチャ用）
- 売上計算: 単価 × 総口数 + 事前課金額

---

### 📋 景品設計（競合コピー）タブ

**用途**: 競合の景品明細をコピーして自社設計に編集する **メインの設計画面**

**取込方法（3種類）**:

- **URL貼付**: 「🔗 商品URLを貼って即取込」欄にURL貼付
- **商品Noから**: トレカセンターNo / DOPA / 有料 / 新規 のいずれかから選択
- **他タブから転送**: 🎴商品一覧 / 🎲DOPA一覧 / ✨リライト商品案 で行選択 → 「📋景品設計に転送」

**編集項目**:

- ヘッダ: 単価・総口数・事前課金額・一括上乗せ倍率
- 明細行: カード名・レア・本数・実価値/枚・snkrdunk URL・上乗せ倍率・**発送限**・除外

**⭐ 発送限カラム（重要）**:

- チェックON = そのカードは **必ず物理発送**（実仕入コストで計算）
- チェックOFF = ポイント還元想定（表示PT合計をコストとみなす、最悪ケース）
- 画面上部の「弊社コスト(最悪ケース)」 = 発送限ONの実仕入 + 発送限OFFの表示PT

**🎯 在庫スプシにあるカードは snkrdunkURL＋実価値が自動セット** されます（手入力不要）。

**便利機能**:

- **🔎 URL空欄を自動検索**: URL未入力のカードをスニダンで一括検索
- **🔄 全行の買取価格を取得**: URL入りの全カードの相場を一括取得（結果はカードマスタDBに保存）

---

### ✨ リライト商品案タブ

**用途**: 完売実績商品をベースに、利益率45〜50%になるよう自動調整された100件の下書きを使う

**表示列**:
- No / サムネタイトル / ベースNo（元の完売商品）/ 元タイトル
- 単価(coin) / 総口数 / 総還元率 / 最低保証
- 実利益率 / 上乗せ率 / 調整ステータス / ステータス

**フィルタ**:
- タイトル / No で検索
- 調整ステータス
- 利益率フィルタ（達成 / 不足 / 過剰 / 赤字）

**操作**:
- 行クリック → **「📋 景品設計に転送」** → 全賞構成が景品設計タブに読み込まれる
- 「全列表示」でリライト詳細（設計単価・実仕入・平均上乗せ率など）が見られる

**運用**:
- まず利益率「45-50%(達成)」でフィルタして完成度の高いものから使う
- 「不足」「過剰」は微調整が必要な案

---

### 🖼 カード照合タブ

**用途**: PSA10カードの画像から在庫DBを自動照合してスニダンURLを紐付ける

- 画像アップロード or 既存在庫DBを一括処理
- 名前・カード番号・レアが取れれば自動でスニダンURL候補を提示
- 既に485件超の実績あり

---

### 🎴 トレカセンター商品一覧タブ

**用途**: トレカセンターのDB約9,000件を参照＋最新取込

**主要ボタン**:
- **🔄 トレカセンター販売中の最新を取得**: カテゴリ選択 → 新規追加分がDBに追加
- 新規追加のうち「新規限定」「初回限定」を含むものは **自動で🆕新規ガチャ一覧に振り分け**
- 行クリック → **「📋 景品設計に転送」** で設計タブへ

---

### 🎲 DOPA商品一覧タブ

**用途**: DOPA全件参照＋最新取込＋振り分け編集

**主要ボタン**:
- **🔄 DOPAから最新一覧を取込**: pull_restrictionフラグ判定で自動振分け（🆕新規/🎰有料へ）
- **🔧 振り分け編集モード** トグルON → チェックボックスで振り分けを人手修正 → **💾 保存**

---

### 🎰 有料ガチャ一覧タブ

**用途**: 課金条件付きの限定ガチャDB（DOPA同期で自動振分けされたもの）

---

### 🆕 新規ガチャ一覧タブ

**用途**: 登録後X日限定など新規ユーザー向け限定ガチャDB

---

### 📋 商品一覧タブ

**用途**: 自社で設計した商品の管理

**ステータスでフィルタ**:
- 予約中: 設計完了・未販売
- 販売中: 販売開始済み
- 完売: 売切
- ボツ: 採用せず

**操作**:

| ボタン | 遷移 | 在庫への影響 |
|---|---|---|
| ✅ **承認**（予約中限定） | 予約中→販売中 | 予約中数量を販売中数量に移動 |
| ❌ **解除**（予約中限定） | 予約中→ボツ | 引当解除、残数量に戻す |
| 🏁 **完売**（販売中限定） | 販売中→完売 | **数量を実減算（物理在庫減）** |

**運用フロー**:

```
[新規設計/景品設計で保存]
      ↓
  予約中（担当者確認中）
      ↓ [✅承認]
  販売中（実際に販売している）
      ↓ [🏁完売]
  完売（物理在庫が減った）
```

担当者から修正依頼: `予約中 → [❌解除] → ボツ → 作り直し`

---

### 🔄 改善提案タブ

**用途**: 新入荷後、既存の予約中・販売中商品に「差し替えた方が目標に近い」ものがないか自動検出

**使い方**:
1. **「🔄 提案を取得」** ボタン
2. 改善幅の大きい順に提案カードが表示
3. **「✅ 差し替える」** で明細・在庫引当・原価が自動更新

**オプション**:
- 販売中の商品も対象（基本オフ推奨）
- 最小改善幅（提案する閾値）

> 新入荷のたびにチェックする運用がおすすめ。

---

### 📦 在庫タブ

**用途**: 在庫状況を見る＋相場更新＋仕入価格入力＋**URL修復**

**表示列**:

| 列 | 意味 |
|---|---|
| **URL** | ✅個別URL / ⚠️検索一覧URL(要修正) / ❌URL空 |
| 区分 | PSA10 / BOX |
| カード名 / シリーズ / グレード | - |
| 数量 | 物理在庫 |
| 予約中 / 販売中 | 引当中 |
| 残数量 | 新規引当可能枚数 = 数量 − 予約中 − 販売中 |
| 相場 | 1枚あたりの市場相場 |
| 仕入れ価格 | 実仕入コスト（粗利計算に使用） |
| 相場更新 | 最後に相場を更新した日時 |
| snk URL | snkrdunk.comの該当商品URL |
| 引当先 | 引当中の商品IDリスト |

**フィルタ**:
- 区分（PSA10/BOX）/ 残数量あり / 仕入れ価格未入力のみ
- **⚠️URL要修正のみ** ← URL不正の在庫だけを絞込
- カード名検索

**主要ボタン**:

- **📈 全在庫の相場を一括更新**: snkrdunk URL付きの全カードを更新（数分〜）
  - **注: 毎朝06:00 の cron で自動更新されるので、緊急時だけで OK**
- 個別カード選択後:
  - **📈 このカードの相場をsnkrdunkから更新**
  - **💾 仕入れ価格を保存**

**⚠️ URL修復フロー（要修正の在庫がある時）**:

1. 「⚠️URL要修正のみ」フィルタON → 対象カードだけ表示
2. 「🔍 カード名で検索」で対象カードを選ぶ
3. 「⚠️ このカードのスニダンURLが正しく設定されていません」の警告が出る
4. **「🔍 候補を検索」** → スニダン公式検索が動く
5. 候補一覧から正しいカードをラジオボタンで選択
6. **「💾 このURLで保存」** → 在庫スプシに保存
7. 「🔗 選択候補を開く」で事前に snkrdunk 側で正しさを目視確認可能

---

### ⚙️ 上乗せ率設定タブ

**用途**: 価格帯ごとのコイン上乗せ率とプリセットの編集

**編集方法**:
1. 上乗せ率テーブルで価格下限・上限・上乗せ率を編集
2. **💾 設定を保存** → 即反映
3. 下部「テスト用相場」で動作確認できる

**プリセット**: 標準（全等1.5倍）/ 高還元型 / 攻めの設計 / 薄めバランス / 限定ガチャ型 など、名前付きで保存された配分パターンを新規設計・景品設計で呼び出せます。

**注意**: 上乗せ率を変えると **過去商品の顧客還元率の表示も変わります**（再計算される）。一度販売した商品の値段は変えないでください。

## 6. 計算結果の見方（画面上部の固定表示）

**景品設計タブ・限定ガチャタブ**は、明細を編集すると **画面上部に計算結果が固定表示** されます。

### 1段目

| 指標 | 意味 |
|---|---|
| **売上** | 単価 × 総口数 + 事前課金額 |
| **表示ポイント数(発行総額)** | 全カードの表示PT合計 = ポイント還元された場合の最悪コスト |
| **弊社コスト(最悪ケース)** | 発送限ON=実仕入 + 発送限OFF=表示PT還元想定 の合計 |
| **弊社の得** | 売上 − 弊社コスト(最悪ケース) と 利益率 |

### 2段目（サマリ）

- 実仕入合計 / 実還元率 / 顧客還元率 / 上乗せ差分 など

### 警告

- 🔴 致命的（赤字確定・在庫不足で組めない等）→ 必須対応
- 🟡 注意（乖離大・バラつき大等）→ 確認推奨
- 🔵 情報（参考メモ）→ 任意

## 7. 用語辞典

| 用語 | 意味 |
|---|---|
| **通常ガチャ** | pt消費型ガチャ（DOPA/トレカセンター共通の基本形式） |
| **新規ガチャ** | 登録後X日限定 / 24時間限定など、新規ユーザー限定 |
| **有料ガチャ** | ○○円課金者だけ引ける条件付き限定ガチャ |
| **引く権利の事前課金額** | 有料ガチャを引くために事前に必要な課金額（円） |
| **売上計算** | 単価 × 総口数 + 引く権利の事前課金額（有料時のみ加算） |
| **実価値/枚** | スニダン買取価格ベースの「運営の実コスト」 |
| **顧客還元率** | 表示PT合計 ÷ 売上（顧客が見る還元率） |
| **実還元率** | 仕入れ合計 ÷ 売上（運営の本当の還元率） |
| **上乗せ差分** | 顧客還元率 − 実還元率（コイン上乗せで顧客に見える還元率を盛る差分） |
| **発送限** | チェックON=物理発送確定・実仕入コスト計上、OFF=ポイント還元想定 |
| **弊社の得（利益）** | 売上 − 弊社コスト(最悪ケース) |
| **粗利率** | 弊社の得 ÷ 売上 = 1 − 実還元率 |
| **乖離率** | 目標相場と実カード相場の差の割合（🟢±10% / 🟡±10-30% / 🔴±30%超） |
| **残数量** | 数量 − 予約中 − 販売中 = 新規引当可能枚数 |
| **リライト商品案** | 完売実績商品をベースに利益率45-50%で自動調整された100件の下書き |

## 8. スプシを触るときのルール

このツールは在庫管理スプシ（ポケモン在庫管理）の以下に自動で書き込みます。手動編集すると数値がズレるので必ず守ってください。

### 🚫 手動編集NG（ツール管理列）

#### PSA10在庫登録 / ボックス在庫記録 タブ

| 列名 | 管理内容 |
|---|---|
| 予約中数量 / 販売中数量 / 残数量 | 引当計算が壊れる |
| 引当ステータス / 引当先商品ID | 商品追跡不能に |
| 最終更新日時 / 相場更新日時 | ツールが自動記録 |

#### 商品設計 / 商品設計明細 タブ（全列）

商品ID・タイトル・ステータス・還元率など。すべてツール管理。
**「完売にしたい」「解除したい」は必ずツールのボタンから操作** してください。

### ✅ 手動編集OK

新カードの仕入れや相場更新は以下を手動で変えてOK（ただしツール経由推奨）:

| 列名 | 編集タイミング |
|---|---|
| カード名 / 英語名 / シリーズ / グレード / カード番号 / PSA Cert# | 情報追記・修正 |
| 数量 | **新規仕入れ時のみ** |
| 相場（1枚） | 推奨はツールの「📈 相場更新」経由 |
| 仕入れ価格 | 推奨はツールの「💾 仕入れ価格を保存」経由 |
| 画像URL | 画像追加時 |
| スニダン used URL | 推奨はツールの在庫タブ「🔍 候補を検索」経由 |

### 📦 新規仕入れ時の正しい手順

1. スプシの新しい行を追加（または既存行の「数量」を増やす）
2. **同じ行の「残数量」も必ず同じ数増やす**
   - 例: 数量 5 → 8 に増やしたら、残数量も 5 → 8
3. 予約中数量・販売中数量は 0 のまま
4. ツールを再読込（「🔄 再読込」ボタン or ページリロード）
5. スニダンURLはツールの在庫タブ「🔍 候補を検索」で入れる

> 面倒な場合は、新規仕入れも運用担当者に依頼するのが安全です

### ❌ やってはいけないこと

- 商品設計タブの「ステータス」を手動で「完売」に変える
- 予約中数量を手動で0にする
- 引当先商品IDを手動削除
- 商品設計明細の行を手動削除

### 😰 間違って編集してしまった時

1. スプシの「ファイル」→「変更履歴」→「版の履歴を表示」で編集前に戻せます
2. それでもダメな場合は開発者に連絡

## 10. よくある困りごとQ&A

### Q. 相場更新バナーが「❌ 要確認」（48時間超）

→ 毎朝の自動更新が失敗している可能性。開発者へ連絡してください。

### Q. 画面上部の計算結果が「-」表示

→ カード明細のどれかで実価値/枚が0になっている可能性。除外チェックを見直すか、URLから相場取得してください。

### Q. 警告で「在庫不足」と出た

→ 以下のどれか:
1. 仕入れて補充する
2. 当たり数を減らす
3. 目標相場を変えて別のカードで賄う
4. 他商品の予約を解除して回す

### Q. 保存後に等構成を変えたい

→ 📋商品一覧で該当商品を **❌ 解除** → ボツ化して在庫を戻す → 新規設計で作り直し

### Q. 販売中の商品を中止したい

→ 現状、販売中→ボツへは直接戻せません。開発者に連絡してください（手動処理推奨）。

### Q. 同じカードが複数商品で予約中に？

→ なりません。引当中カードは「残数量=0」扱い。複数数量ある在庫（例:BOX 数量10）は3個予約しても残7個が別商品で使えます。

### Q. 新カード仕入れたのに候補に出てこない

→ スプシの「残数量」列が正しいか確認。数量を増やしたら残数量も同じ数増やす必要があります。ツール側で「🔄 再読込」で反映。

### Q. スプシを見たら数値がおかしい

→ 手動編集で「ツール管理列」に触っていないか確認。予約中数量・販売中数量・残数量・引当ステータスは触らないでください。

### Q. 赤字警告が消えない

→ 以下のいずれか調整:
- 1回価格を上げる / 総口数を増やす
- 各等の目標相場を下げて原価圧縮
- 当たり数を減らす

警告メッセージ内に具体的な推奨値が出ます。

### Q. 「gspread.exceptions.APIError」が出る

→ Sheets API のクォータ瞬間超過。リロードすれば数秒〜数分で復活します（自動リトライ内蔵）。頻発時は時間を置いて再試行。

### Q. 在庫にあるカードなのにスニダンURLが入らない

→ 在庫スプシのカード名と景品明細のカード名の表記揺れの可能性。名前の一部・空白・全半角の違いを確認してください。

### Q. URLが「⚠️」になっている在庫がある

→ 在庫タブで「⚠️URL要修正のみ」ON → 対象カード選択 → 「🔍 候補を検索」→ 候補選択 → 「💾 このURLで保存」で修復できます。

### Q. 振り分け（新規限定/有料）が違う

→ 🎲DOPA商品一覧の **🔧振り分け編集モード** で人が修正できます。
