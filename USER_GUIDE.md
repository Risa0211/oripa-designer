# みんなのトレカ オリパ商品設計ツール 使い方ガイド

## ⚠️ 重要: このツールとスプレッドシートの関係

このツールは **Google スプレッドシート「ポケモン在庫管理」と常時連動** しています。

- ツールで操作した内容は **即座にスプシに書き込まれます**
- 逆にスプシを直接編集すると、ツール側の表示も変わります
- 下記「スプシを手動で編集する時の注意」を必ず読んでください ⚠️

---

## このツールでできること

競合の人気オリパを参考にしながら、自社の在庫から自動でカードを選定して商品設計ができます。勘に頼らず、データに基づいた商品作りが可能です。

### 主な機能

1. **競合オリパを参考に設計**：日本トレカセンターの完売オリパ8,000件超のデータから模倣したいものを選択
2. **自動マッチング**：目標相場に合うカードを在庫から自動選定
3. **手動調整**：自動選定結果をドラッグ感覚で追加・削除
4. **引当管理**：予約中→販売中→完売の状態遷移を自動で在庫反映
5. **改善提案**：新入荷でより良いカードが入ったら自動で差し替え提案

## ログイン

1. 共有されたURLをブラウザで開く
2. パスワードを入力
3. 「ログイン」ボタン

> パスワードが分からない場合は運用担当者に確認してください。

## 📝 新規設計の流れ

### ステップ1: 参考競合を選ぶ（左サイドバー）

1. 「検索」欄にキーワード入力（例：「ブラッキー」「おつきみ」「1758」）
2. ドロップダウンから競合オリパを選択
3. 下部に競合の情報（1回価格・総口数・等構成）が表示される
4. 「等構成（参考）」で各等にどんなカードが入っているか確認できる

### ステップ2: 販売パラメータを入力

| 項目 | 意味 | 例 |
|---|---|---|
| **総口数** | 全部で何口売るか | 7,000 |
| **1回価格** | 1回あたりの販売価格（円） | 999 |
| **目標粗利率** | 売上のうち利益として残したい割合 | 30% |
| **還元率** | 原価として使える割合（=100-粗利率、自動計算） | 70% |

> 初期値は選択した競合の値が自動セットされます。

### ステップ3: モードを選ぶ

| モード | 使いどころ |
|---|---|
| **案X: 1枚あたり目標相場を直接指定** | 「1等は¥200,000クラス」のように具体的な相場イメージがある時 |
| **案Y: 還元率＋等級配分比率から自動計算** | 還元率と等ごとの予算配分比率を入れて自動で目標算出 |

> どちらでも構いません。慣れてきたらお好みで使い分けてください。

### ステップ4: 等構成を設定

1. 「含める等」でどの等を作るか選択（初期値は競合と同じ）
2. 各等について以下を入力
   - **当たり数**：その等の当選枚数（初期値は競合と同数）
   - **1枚あたり目標相場**（案X） or **原価配分比率**（案Y）

### ステップ5: 自動提案を取得

**「🔍 自動提案」ボタン** をクリック

- 在庫から目標相場に最も近いカードが自動で選定されます
- 各等が展開されて選定結果が表示される
- 上部に総売上・原価合計・実還元率・実粗利率が表示される
- 警告があれば折りたたまれて表示（クリックで展開）

### ステップ6: 手動で調整（必要に応じて）

各等の展開パネル内で：

- **❌ ボタン**：選定されたカードを削除
- **📋 候補カード**：目標に近い未使用の在庫から追加できる
  - フィルタ：乖離範囲（±10% / ±25% / ±50% / 全て）、区分（PSA10/BOX）、表示件数
  - **➕ ボタン**：そのカードを等に追加
- 追加・削除するたびに原価・還元率・警告が自動再計算

### ステップ7: 保存

**「💾 この内容で保存」** をクリック

- 商品ID（自動発番）が発行される
- 在庫の該当カードが「予約中」に変わる
- 商品設計タブに行が追加される

## 各数値の意味

### 目標相場（案Xモード）

各等の当選カード1枚あたり、いくらぐらいのカードを配置したいか。競合オリパの等構成を見て、「このレベルのカードだから¥〜くらいだろう」と推定して入力する値。

### 2種類の還元率（重要）

このオリパの仕組みは「ゲーム内コインで上乗せ」する形式のため、**還元率が2種類**存在します。

| 還元率 | 意味 | 計算式 |
|---|---|---|
| **顧客還元率** | 顧客が見る還元率（コイン額面ベース） | コイン合計 ÷ 売上 |
| **実還元率** | 運営の本当の還元率（仕入れベース） | 仕入れ合計 ÷ 売上 |

例: 相場¥100,000のカードに20%上乗せ → コイン額面120,000コイン
- 顧客には「120,000コイン分還元」と表示
- 運営の実コストは仕入れ¥80,000（と仮定）→ 実還元率は80%相対比でずっと低い

### 上乗せ率

カード相場にどれだけコインを上乗せするか。価格帯ごとに異なる率を設定できる（**⚙️ 上乗せ率設定**タブで編集）。

例: 
- ¥10,000未満 → 10%上乗せ
- ¥10,000-100,000 → 15%上乗せ  
- ¥100,000-1,000,000 → 20%上乗せ
- ¥1,000,000以上 → 25%上乗せ

### 粗利率

= (売上 − 仕入れ合計) ÷ 売上 = 1 − 実還元率

**仕入れ価格が未入力**のカードは相場を代用するので、正確な粗利を見るには各カードに仕入れ価格を入力してください（**📦 在庫タブ**で操作可）。

### 乖離率

目標相場と実際に選ばれたカードの相場との差の割合。

- 🟢 ±10%以内 = ほぼ目標通り
- 🟡 ±10〜30% = 要確認
- 🔴 ±30%超 = 大きく外れている（在庫に合うカードが無い）

### 残数量

その在庫カードで「新しい商品に割り当てられる枚数」。

```
残数量 = 数量（物理在庫） − 予約中数量 − 販売中数量
```

## 警告の見方

「⚠ 警告・提案」を展開すると、カテゴリ別に警告が表示されます。

| アイコン | 意味 | 対応優先度 |
|---|---|---|
| 🔴 | 致命的（赤字確定・在庫不足で組めない等） | 必須対応 |
| 🟡 | 注意（乖離大・バラつき大等） | 確認推奨 |
| 🔵 | 情報（参考メモ） | 任意 |

### 警告のカテゴリ

| カテゴリ | 内容 |
|---|---|
| 利益（計算結果） | 赤字リスク・目標粗利に届かない等の計算結果ベース |
| 目標価格帯の在庫プール | 目標±10%帯に何枚あるか、豊富/薄い/枯渇 |
| 等内マッチング品質 | 相場バラつき、外れ値、同一カード重複、シリーズ偏り |
| 価格帯消費率 | この商品で¥X万円帯を何%使う、他商品への影響 |
| 在庫不足 | 要求数に対し確保できた数 |
| 競合パターン整合 | BOX型競合なのにBOX未使用等 |

### 警告への対応

- 警告は**全部消す必要はありません**。商品として成立するかの判断材料です。
- 赤字確定（🔴）は必須対応、それ以外は運用方針に合わせて判断。
- 提案内容（💡）を読んで、目標相場・当たり数・口数・価格の調整を検討してください。

## 📋 商品一覧タブ

作成済みの商品が一覧表示されます。

### フィルタ

「ステータスでフィルタ」で表示する状態を選択：
- **予約中**：設計完了、まだ販売していない
- **販売中**：販売を開始した
- **完売**：売切れた
- **ボツ**：設計したが採用しなかった

### 操作

商品IDを選択すると、以下の操作が可能：

| ボタン | 遷移 | 在庫への影響 |
|---|---|---|
| ✅ **承認**（予約中限定） | 予約中 → 販売中 | 予約中数量を販売中数量に移動 |
| ❌ **解除**（予約中限定） | 予約中 → ボツ | 引当解除、残数量に戻る |
| 🏁 **完売**（販売中限定） | 販売中 → 完売 | **数量を実減算（物理在庫減）** |

### 運用の流れ

```
[新規設計で保存]
      ↓
  予約中（担当者確認中）
      ↓
  [承認ボタン]
      ↓
  販売中（実際に販売している）
      ↓
  [完売ボタン]
      ↓
  完売（物理在庫が減った）
```

担当者から修正依頼があったら：
```
予約中 → [解除] → ボツ
       ↓
  新規設計で作り直し
```

## 🔄 改善提案タブ

**使いどころ**: 新しいカードを仕入れた後、既に予約中・販売中の商品の中に「このカードに差し替えた方が目標に近い」ものがないか自動チェック。

### 使い方

1. 「🔄 提案を取得」ボタン
2. 各提案カードが改善幅の大きい順に表示
3. 内容を確認して **「✅ 差し替える」** ボタン
4. 明細・在庫引当・原価合計が自動で更新される

### オプション

- **販売中の商品も対象**：チェックを入れると販売中も候補に（基本オフ推奨）
- **最小改善幅**：どれくらい良くなれば提案するかの閾値

> 定期的（新入荷のたび）にチェックする運用がおすすめ。

## 📦 在庫タブ

現在の在庫状況を確認＋**相場更新・仕入れ価格入力**ができます。

### 表示される列

| 列 | 意味 |
|---|---|
| 区分 | PSA10 / BOX |
| カード名 | - |
| 数量 | 物理在庫（手元にある枚数） |
| 予約中 | 仮引当中 |
| 販売中 | 販売中の商品で使用中 |
| 残数量 | 数量 − 予約中 − 販売中 = 新規引当可能枚数 |
| 相場 | 1枚あたりの市場相場（snkrdunkから取得・更新可） |
| 仕入れ価格 | 実際の仕入れコスト（粗利計算に使用） |
| 相場更新 | 最後に相場を更新した日時 |
| snk URL | snkrdunk.comの該当商品URL |
| 引当先 | 引当中の商品IDリスト |

### 📈 相場更新

- **「全在庫の相場を一括更新」**: snkrdunk URL付きの全カードを自動更新（数分〜）
- **個別更新**: カードを選択 → 「このカードの相場をsnkrdunkから更新」
- 取得元: snkrdunk.com の **直近1個販売価格**（無ければ中古最安）

### 💰 仕入れ価格入力

下部の「個別操作」セクションから、選択したカードに仕入れ価格を入力。

> 仕入れ価格は粗利計算の基準値です。正確な経営判断のため、商品仕入れ時に必ず入力してください。

### フィルタ

- 区分（PSA10/BOX）
- 残数量あり
- 仕入れ価格未入力のみ（一括入力作業に便利）
- カード名検索

## よくあるケースと対応

### Q. 警告で「在庫不足」と出た

→ 以下のどれかを選択：
1. 仕入れて補充する
2. 当たり数を減らす
3. 目標相場を変えて別のカードで賄う
4. 他商品の予約を解除して回す

### Q. 商品を一度保存したが、等構成を変えたい

→ 商品一覧で該当商品を「❌ 解除」→ボツ化して在庫を戻す → 新規設計で作り直し

### Q. 販売中の商品を中止したい

→ 現状、販売中からボツへは直接戻せません。もし必要であれば開発者に連絡してください（データ整合性のため手動処理推奨）

### Q. 同じカードが複数の商品で予約中になる？

→ なりません。引当中のカードは「残数量 = 0」扱いになり、他の商品では選ばれません。複数数量ある在庫（例：BOX 数量10）は、3個予約しても残7個は別商品で使えます。

### Q. 新カード仕入れたのに候補に出てこない

→ 在庫スプシの「残数量」列が正しく入っているか確認。仕入れ時は「数量」を増やした上で「**残数量も同じ数に増やす**」必要があります（予約中・販売中は0のまま）。その後、ツールの「🔄 再読込」ボタン or ページリロードで反映されます。

### Q. スプシを見たら数値がおかしい

→ ツールで操作した内容とスプシを手動で編集した内容が衝突していないか確認。手動編集OKなのは「数量」「相場」「カード名」等の **人間が管理する列** のみ（本ガイド上部「⚠️ スプシを手動で編集する時の注意」参照）。予約中数量・販売中数量・残数量・引当ステータスは **ツール専用** です。

### Q. 赤字警告が消えない

→ 以下のいずれかを調整：
- 1回価格を上げる
- 総口数を増やす
- 各等の目標相場を下げて原価圧縮
- 当たり数を減らす

警告メッセージ内に**具体的な推奨値**が表示されるので、それを参考に調整してください。

## ⚙️ 上乗せ率設定タブ

価格帯ごとのコイン上乗せ率を編集できます。

### 編集方法

1. ⚙️ **上乗せ率設定タブ**を開く
2. テーブルで価格下限・上限・上乗せ率を編集
3. 行追加: 一番下の空行に入力
4. 行削除: 行を選択してDelete
5. **「💾 設定を保存」** クリック → 即反映

### 動作確認

下部の「テスト用相場」に金額を入れると、その価格に該当する上乗せ率とコイン額面を確認できます。

### 注意

- 上乗せ率を変更すると、**過去の商品の顧客還元率の見え方も変わります**（再計算される）
- 一度販売した商品の値段は変えないでください

## ⚠️ スプシを手動で編集する時の注意

このツールは、在庫管理スプシ（ポケモン在庫管理）の以下に自動で書き込みます。
**スプシ側で手動編集すると数値がズレて大変なことになる**ため、注意してください。

### 🚫 手動編集NG（ツールが管理している列・タブ）

以下はツールが自動で書き込み・更新します。**絶対に手で触らない**でください。

#### PSA10在庫登録 / ボックス在庫記録 タブの以下の列

| 列名 | 管理内容 |
|---|---|
| **予約中数量** | ツールで予約した商品の数（手で変えると残数量計算が狂う） |
| **販売中数量** | 販売中の商品で使用中の数 |
| **残数量** | 数量 − 予約中 − 販売中（自動計算） |
| **引当ステータス** | 「予約中x3 / 販売中x2 / 残5」等の集計表示 |
| **引当先商品ID** | どの商品で予約中かのリスト |
| **最終更新日時** | ツールが最後に更新した時刻 |
| **相場更新日時** | snkrdunkから相場を取得した日時 |

#### 商品設計 タブ（全列）

商品ID・タイトル・ステータス・還元率など。すべてツール管理。
**「完売にしたい」「解除したい」は必ずツールのボタンから操作**してください。

#### 商品設計明細 タブ（全列）

各商品がどのカードを使っているかの記録。すべてツール管理。

### ✅ 手動編集OK（人間が管理する列）

新カードの仕入れや相場更新は以下の列を手動で変えてOKです（が、できるだけツール経由推奨）。

| 列名 | 編集タイミング |
|---|---|
| **カード名** | 新カード追加時 |
| **英語名 / シリーズ / グレード / カード番号 / PSA Cert#** | 情報追記・修正 |
| **数量** | **新規仕入れ時のみ**（数量を増やす）⚠️ |
| **相場（1枚）** | 推奨はツールの「📈 相場更新」ボタン経由 |
| **仕入れ価格** | 推奨はツールの「💾 仕入れ価格を保存」経由 |
| **画像URL** | 画像追加時 |
| **スニダン used URL** | snkrdunkの該当商品URL（相場自動取得に必須） |

### 📦 新規仕入れ時の正しい手順

仕入れて在庫を増やす場合：

1. スプシの新しい行を追加（または既存行の「数量」を増やす）
2. 同じ行の **「残数量」も必ず同じ数増やす** ⚠️
   - 例: 数量 5 → 8 に増やしたら、残数量 も 5 → 8 に増やす
3. 予約中数量・販売中数量は **0** のまま（触らない）
4. ツールを再読込（「🔄 再読込」ボタン or ページリロード）

> 💡 面倒な場合は、新規仕入れも運用担当者に依頼するのが安全です

### ❌ やってはいけないこと

- スプシで商品設計タブの「ステータス」を手動で「完売」に変える
  - → ツール経由で「完売」ボタンを押さないと、数量が減りません
- 予約中数量を手動で0にする
  - → 実際には予約中だった商品との整合性が取れなくなる
- 引当先商品IDを手動削除
  - → どの商品が何を使っているかの追跡不能になる
- 商品設計明細の行を手動削除
  - → その商品の構成が失われ、完売・解除操作が壊れる

### 😰 間違って編集してしまった時

1. スプシの「ファイル」→「変更履歴」→「版の履歴を表示」で編集前に戻せます
2. それでもダメな場合は開発者に連絡
