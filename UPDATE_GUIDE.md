## 1. 還元率の二系統化（顧客還元率と実還元率）

### 概要

オリパは「ゲーム内コインで上乗せして売る」仕組みのため、**還元率が2種類**存在します。

| 還元率 | 意味 | 計算式 |
|---|---|---|
| **顧客還元率** | 顧客が見る還元率（コイン額面ベース） | コイン額面合計 ÷ 売上 |
| **実還元率** | 運営の本当の還元率（仕入れベース） | 仕入れ合計 ÷ 売上 |
| **粗利率** | 利益として残る割合 | (売上 − 仕入れ合計) ÷ 売上 |

### 表示位置

**📝 新規設計タブ** / **🎰 限定ガチャタブ** で「🔍 自動提案」を押すと、結果上部に以下が表示されます。

```
| 売上     | 仕入れ合計 | 粗利 (粗利率%) |
| ¥X,XXX   | ¥X,XXX    | ¥X,XXX (XX%)  |

| 顧客還元率 | 実還元率 | 上乗せ差分    |
| XX%       | XX%     | +X.X pt      |
```

### ポイント
- **顧客にはコインの額面で「還元率○○%」と見せる**（高めに見える）
- **運営の実コストは仕入れ価格ベース**で、粗利確保はこちらで判断
- 「上乗せ差分」が顧客還元率 − 実還元率の差（コイン上乗せで稼いだ見せかけの還元率）

## 2. snkrdunk相場の自動取得

### 概要

在庫スプシの「スニダン used URL」列にあるURLから、**直近の販売価格（PSA10など同グレード優先）** を自動取得して相場列を更新。

### 使い方

**📦 在庫タブ**の上部に2つのボタン:

- **📈 全在庫の相場を一括更新**: snkrdunk URL付き全カードを順番に更新
- **個別更新**: 下部の「個別操作」セクションで対象カードを選択 → **📈 このカードの相場をsnkrdunkから更新**

### 取得ロジック

1. **直近グレード一致販売** (例: PSA10カードならPSA10販売の直近価格)
2. グレード一致なし → サイズ "1個/1枚" の直近販売
3. 履歴なし → 中古最安出品価格
4. それでもなし → 新品最安出品

スプシには相場と「相場更新日時」「取得元」が記録されます。

### 注意

- 在庫スプシの **「スニダン used URL」** はカード番号と完全一致するURLを入れる
- 同名カードでもカード番号（XY-P 203 vs 230等）違いは別商品
- 違うURLだと別カードの相場を取ってしまうので、現物確認が大事

## 3. 仕入れ価格管理

### 概要

「相場」とは別に「仕入れ価格」を管理。**粗利計算は仕入れ価格ベース**で算出されます。

### 入力方法

**📦 在庫タブ**の下部「個別操作」セクション:

1. **🔍 カード名検索** で対象カードを絞り込み
2. プルダウンから1枚選択
3. **「仕入れ価格（円）」** に金額入力
4. **💾 仕入れ価格を保存**

### 一括入力のヒント

「仕入れ価格未入力のみ」フィルタを使うと未入力カードだけ抽出できます。

### 未入力時の挙動

仕入れ価格が空のカードは、**相場を仕入れ価格として代用** して粗利計算されます（警告表示あり）。正確な経営判断のため、商品仕入れ時には必ず入力してください。

## 4. コイン上乗せ率（3層構造）

### 概要

カード相場 → コイン額面への上乗せルール。**3層で柔軟に管理**できます。

### 優先順位

```
等別 > 商品全体ベース倍率 > 価格帯別ルール（⚙️タブ）
```

| 層 | 設定場所 | スコープ | 例 |
|---|---|---|---|
| **等別** | 商品設計時の各等 | この商品のこの等だけ | 1等 = 100% (2倍) |
| **商品全体ベース** | 商品設計画面の頭 | この商品の全等共通 | ベース 50% (1.5倍) |
| **価格帯別** | ⚙️タブ（全商品共通） | 全商品の一般ルール | 10万円以上 = 20% |

### 各層の動作

**等別** に `0以上` の値を入れる → その値を最優先
**等別** が `-1` → 商品全体ベース倍率を使用
**商品全体ベース** が `-1` → 価格帯別ルールにフォールバック

### よくある運用

- **時短設計**: 商品全体ベース「50%」だけ設定（全等1.5倍）→ そのまま使う
- **メリハリ設計**: ベース「50%」+ 1等だけ等別「100%」（1等は2倍、それ以外1.5倍）
- **完全カスタム**: 全等を等別で個別指定

## 5. 上乗せ率プリセット

### 概要

よく使う上乗せ率パターンを **名前付きで保存** → ワンクリック適用。

### デフォルトプリセット

| プリセット名 | ベース率 | 等別上書き |
|---|---|---|
| **標準（全等1.5倍）** | 50% | なし |
| **高還元型** | 0% | なし（顧客還元率=実還元率） |
| **攻めの設計** | 50% | 1等100%, 2等75%, 3等50%, ラストワン100% |
| **薄めバランス** | 20% | なし |
| **限定ガチャ型** | 30% | S/A/B/C/D賞ごとに段階的 |

### 使い方

**📝 新規設計 / 🎰 限定ガチャ** の「🎯 商品全体の上乗せ率設定」セクション:

1. **プリセット ドロップダウン** から選択
2. **✅ プリセット適用** クリック
3. ベース倍率と等別の値が一括反映される
4. その後、必要なら手動で微調整

### プリセット追加・編集

**⚙️ 上乗せ率設定タブ**の下部「📋 上乗せ率プリセット」セクション:

- 既存プリセットを直接編集
- 行追加でオリジナルプリセット作成
- **💾 プリセットを保存** で全件反映

## 6. 無在庫モード

### 概要

実際の在庫に縛られず、**全カードから自由に選んで商品設計**するモード。販売決定後に仕入れる運用向け。

### 切替方法

**📝 新規設計タブ** の「② 在庫モード」で **「無在庫」** を選択（既定は「在庫連動」）。

### 動作

| 項目 | 在庫連動モード | 無在庫モード |
|---|---|---|
| 候補カード | 残数量がある在庫のみ | 全カード（残数量0でも） |
| 保存時の在庫操作 | 予約中数量+N、残数量-N | **操作なし**（スプシ不変） |
| 承認・解除・完売 | 通常通り在庫操作 | **在庫操作スキップ** |
| 商品設計タブの記録 | 通常通り保存 | メモ欄に【無在庫】タグ自動付与 |

### 使いどころ

- 大手競合の模倣でとりあえず商品設計しておく
- 仕入れ判断の前にシミュレーション
- 在庫薄いカードも含めて全部の選択肢から比較したい

## 7. 限定ガチャ設計（🎰 新タブ）

### 概要

DOPA型の限定ガチャ専用設計タブ。通常ガチャと違って以下の特徴があります:

- **当たり枠** + **外れ枠（ポイント還元）** + **ラストワン賞**
- **最低保証ポイント**（外れでも最低◯ptもらえる）
- **ポイント実コスト率**（運営の経験値で実コスト換算）

### 入力項目

| 項目 | 説明 |
|---|---|
| 在庫モード | 在庫連動 / 無在庫 切替 |
| 総口数 / 1回コイン消費（pt） | 売上の基準 |
| 目標粗利率（%） | 経営目標 |
| 商品全体ベース上乗せ率 / プリセット | コイン上乗せ |
| 最低保証pt | 外れ枠の残口数に配るpt |
| ポイント実コスト率（%） | デフォルト70%、塩漬け率を反映 |
| 当たりカード等構成 | S/A/B/C賞... 自由に追加削除 |
| 外れポイント枠 | 5,000pt×5口、3,000pt×15口...のような区分指定 |
| ラストワン賞 | カード or ポイント（オプション） |
| 競合参考 | 既存リサーチDBから検索可能 |

### ポイント実コスト率とは

還元したポイントのうち**実際にコスト化する割合**:

- **100%**: 全部消費される（最悪ケース、完全コスト化）
- **70%**: 平均的（運営の経験値依存、初期値）
- **0%**: 全部塩漬けで消費されない（最良）

これが運用の肝で、運営が「ポイントの再利用率」をどう想定するかで実還元率が変わります。

### 結果表示

```
売上     | 実コスト        | 粗利 (粗利率)
顧客還元率 | 実還元率        | コイン上乗せ差分

詳細内訳:
- 売上                       ¥XX,XXX,XXX
- カード相場合計             ¥XX,XXX,XXX
- カード仕入れ合計           ¥XX,XXX,XXX
- ポイント還元（額面）        ¥XX,XXX,XXX
- ポイント還元（実コスト×70%）¥XX,XXX,XXX
- コイン額面合計（顧客視点）  ¥XX,XXX,XXX
- 実コスト合計               ¥XX,XXX,XXX
- 粗利                       ¥X,XXX,XXX
```

### 口数の整合性

総口数 = 当たりカード口数 + 外れポイント口数 + ラストワン1口

設定が合わないと「総口数の整合性」警告が表示されます。

### 商品一覧での見分け

限定ガチャは商品設計タブで **モード列が「Premium」**、メモ欄に **【限定ガチャ】** タグが入ります。

## 9. テストモード

### 概要

本番在庫スプシではなく **テスト用スプシ** に向けて、影響を出さずに動作確認できます。

### 切替

ロゴ直下のトグル **「🧪 テストモード」** で切替。

- **OFF (既定)**: 本番モード - 本番の在庫スプシに反映
- **ON**: テストモード - テスト用スプシ（ポケモン在庫管理（テスト））に反映

### 使いどころ

- 新入りスタッフのトレーニング
- 設定変更や新機能の影響確認
- 担当者にツールを試してもらう時

## トラブル時の対処（追加分）

### Q. 相場更新で「取得失敗」と出る

→ snkrdunk URL が間違っている可能性。商品ページURLが正しいか確認、特にカード番号一致を確認。

### Q. プリセット適用したのに反映されない

→ 商品全体ベース倍率と等別の優先順位を確認。等別に `0以上` の値が残っていれば等別が優先される。

### Q. 限定ガチャの粗利率が想定と違う

→ ポイント実コスト率の設定値を確認。実コスト率を下げるほど粗利率は上がる（運営有利）。

### Q. 顧客還元率が100%を超える

→ コイン上乗せが効きすぎ。ベース倍率を下げるか、当たりカードの仕入れ価格を見直す。100%超は基本ありえない設計なので警告が出る。

## 10. 景品設計タブ（競合コピー型）🆕

### 概要

エクセル「ガチャ設計テンプレート」の運用をツール化。**競合の商品No.を入力すると景品明細が自動展開**され、各カードの本数・実価値・上乗せ倍率を編集して還元率を試算できます。

### 使い方

**📋 景品設計（競合コピー）タブ**:

1. 商品No.を入力（例: `7401`）または 🎰有料ガチャ・🆕新規ガチャから選択
2. **📥 読み込み** ボタン → 景品明細が展開される
3. ヘッダで **単価・総口数・課金額(pt買い増し相当)・一括上乗せ倍率** を調整
4. カード行で各カードの **実価値/枚・snkrdunk URL・上乗せ倍率** を編集
5. 計算結果（売上/仕入れ合計/粗利/顧客還元率/実還元率/上乗せ差分）がリアルタイム表示

### 商品ページURL

商品No.の横に **「商品ページを開く」リンク** が表示されます（japan-toreca.com）。

### 上乗せ倍率の優先

- 各行の「上乗せ倍率」に **0以上の値** → その行は個別倍率
- 各行の「上乗せ倍率」が **0** → ヘッダの「一括上乗せ倍率」を適用

### 課金額（pt買い増し相当）

有料ガチャでは pt買い増し分も売上に含めるため、**売上 = 単価×総口数 + 課金額** で計算されます。

### データ出典

- 商品No.のカード明細 = `data/prize_details.parquet` (304,324行・8,550商品分)
- 明細未登録の商品 = リサーチDB「完売オリパ一覧」の各等列をフォールバック

## 11. スニダン買取価格の自動取得 🆕

### 概要

カード明細の **snkrdunk URL** 列にURLを貼ると、買取価格を取得して **実価値/枚に自動反映**。同時に **カードマスタDB** に保存されるので、次回以降は同じカードが自動でURL+価格付きで読み込まれます。

### 使い方

景品設計タブで景品明細を読み込んだ後:

1. 各カード行の **「snkrdunk URL」列** に商品URLを貼り付け
   - 🔗 **スニダン検索リンク表示** ボタンで各カード名のスニダン検索ページを開ける
   - 商品ページに辿り着いたら、その URL をコピーして貼る
2. **🔄 全行の買取価格を取得** ボタン → 実価値が自動入力される
3. 次回同じカードが出てきた時はマスタDB経由で自動入力

### マスタDB

**リサーチDB の「カードマスタ」タブ**に蓄積:
- カード名 / レアリティ / snkrdunk URL / 買取価格(円) / 取得元 / 価格更新日時

蓄積されるほど自動入力率が上がります。

## 12. 🎰 有料ガチャ一覧タブ 🆕

### 概要

DOPA / DOKKAN / EXTRECA / JTC などの**有料ガチャ**を一覧管理。「📋 景品設計」タブから選択して設計に流用できます。

### 機能

- **🔄 DOPAから最新一覧を取込** ボタン: DOPA(ポケモン)の全ガチャを自動取得（約1-2分）
  - 新規ガチャ判定（タイトルに「新規登録から○日限定」等を含むもの）も同時に🆕タブへ振り分け
- 手動追加: 商品ID/サイト/タイトル/単価/総口数/カード種数/課金額/URL/備考
- 削除: 商品IDを選んで削除

### 課金額の運用

DOPAのような pt買い増し（=課金）がある有料ガチャでは「課金額(pt買い増し相当)」を入力。「📋 景品設計」タブで読み込むと売上に自動加算されます。

### DOPA同期の仕組み

`dopa_scraper.py` がDOPAサイトの `/pokemon/gacha/{ID}` ページから og:title を読み取り、リサーチDBの「有料ガチャ一覧」タブにupsertします。価格・賞構成・カード明細は **SPA描画のため自動取得不可** → タイトル・URL・商品IDのみ自動、詳細は手動補完。

## 13. 🆕 新規ガチャ一覧タブ

### 概要

**トレカセンター登録後X日限定**などの新規ガチャを管理。

### 自動候補 + 人による確認

DOPA同期時、タイトルに以下のパターンを含むものを自動で新規ガチャとして候補登録します:
- 「新規登録から24時間限定」
- 「新規登録から7日限定」
- 「新規限定」「初回限定」「はじめて限定」

候補は **備考に「自動候補(タイトルから判定)」** と入るので、人が確認して必要なら削除・修正します。

### 手動追加

トレカセンター由来の新規ガチャは商品No.を入力して登録（**景品明細から自動展開可**）。
